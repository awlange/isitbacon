State-of-the-art bacon detection technology

About BaconNet

BaconNet is a deep convolution neural network trained to recognize bacon, Kevin Bacon, and things that are not bacon. The aim of BaconNet is not so much to make the best model for detecting these categories, but it is more a display of building a model from scratch and turning it into a product. BaconNet was built as follows:

  • Gather images from Google: 100 bacon images, 100 Kevin Bacon images, 200 semi-random images
  • Transform original images into 64x64 pixel thumbnails (ignoring/allowing skewing)
  • Generate more data by vertical reflection and single pixel translations in ech cardinal direction, amounting to 10x as much data as original (4000 images)
  • Split data by 80:10:10 of train:validation:test
  • Using Lasagne, build and train network with architecture:
    1. Input layer (3 RGB channels, 64x64 pixels)
    2. Convolution Layer: 32 16x16 pixel filters, ReLU
    3. Max Pool Layer: 2x2 pool
    4. Convolution Layer: 32 4x4 node filters, ReLU
    5. Max Pool Layer: 2x2 pool
    6. Fully Connected Layer: 512 nodes, sigmoid logistic units, 0.2 dropout
    7. SoftMax Layer: 3 nodes
  • Training run with combination of stochastic gradient descent followed by RMSprop
  • Resulting accuracy of ~85% on test data (not great, so don't be surprised by misclassification!)
  • Web app built with Flask and stuff
